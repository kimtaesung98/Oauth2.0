# 새 폴더 생성 체크리스트

> 에이전트 또는 개발자가 새 폴더를 생성할 때 이 체크리스트를 순서대로 완료하세요.

---

## 폴더 생성 시 필수 작업

- [ ] `templates/README_TEMPLATE.md`를 복사해 새 폴더에 `README.md`로 저장
- [ ] `README.md`의 `[폴더명]` 헤더를 실제 폴더명으로 교체
- [ ] Overview 섹션을 이 폴더의 책임 범위에 맞게 작성 (1~3문장)
- [ ] DFD를 이 폴더의 실제 데이터 흐름에 맞게 수정
- [ ] Tech Stack을 이 폴더에서 사용하는 기술로 채움
- [ ] Agent Control의 허용/금지/필수 규칙을 이 폴더의 아키텍처 원칙에 맞게 작성
- [ ] Progress Tracker에 초기 기능 목록을 `⏳ Pending`으로 추가
- [ ] Next Roadmap에 첫 번째 작업 항목 기재

---

## 에이전트를 위한 검증 질문

새 폴더의 `README.md`를 작성한 후 아래 질문에 답할 수 있어야 합니다:

1. 이 폴더는 어떤 책임을 가지는가? (Overview)
2. 데이터는 어디서 들어와서 어디로 나가는가? (DFD)
3. 이 폴더에서 절대 해서는 안 되는 일은 무엇인가? (Agent Control > 금지)
4. 현재 구현된 기능과 미구현 기능은 무엇인가? (Progress Tracker)
5. 다음에 해야 할 일은 무엇인가? (Next Roadmap)

이 다섯 가지 질문에 답할 수 없다면 `README.md`가 불완전한 것입니다.
